MOOCAMS

Another Metadata Schema For MOOCs

MOOCAMS.

  • Title: MOOCAMS
  • Identifier: https://moocams.github.io/
  • Description: A metadata schema for MOOCs

Terms

Term Name: title
URI:https://moocams.github.io/#title
Label:Course Title
Description:The full name given for the course.
Rule:Use the 'take what you see' approach.
Comment:If the course has translated name(s), input the name in original language.
Example:Introduction to Marketing, Introduction to Programming, ...
Term Name: identifier
URI:https://moocams.github.io/#identifier
Label:Course Identifier
Description:The URI of the course.
Rule:Input the complete form of URI that link directly to the page of the course.
scheme://[user:password@]host[:port]][/]path[?query][#fragment]
Comment:URL is prefered. The segments in the brackets are optional.
Example:https://www.coursera.org/learn/python-databases, ...
Term Name: description
URI:https://moocams.github.io/#description
Label:Course Description
Description:The description of the course. It usually includes what the course is about, the basic background information of the content, or a brief introduction to the course.
Rule:Use ‘take what you see’ approach.
Comment:If there is no description available, short description should be provided by the metadata creator.
Example:In this course you will learn how to program in R and how to use R for effective data analysis. You will learn how to install and configure software necessary [...]
Term Name: subject
URI:https://moocams.github.io/#subject
Label:Subject
Description:The area of knowledge of the course.
Rule:Subject is a controlled vocabulary term chosen from Library of Congress Subject Headings (LCSH).
Comment:If the wanted subject is not available in LCSH, you may consider use ‘tags’ element to transcribe the term.
Example:Information science, Python (Computer program language)
Term Name: language
URI:https://moocams.github.io/#language
Label:Language
Description:The language in which the course is taught.
Rule:Language is a controlled vocabulary chosen from ISO 639-3 Code.
Comment:The language should be the language which the instructor(s) use(s) to explain rather than the one in the background or on slides. For instance, even though slides are in English, if the instructor uses Chinese to explain, the language transcribed should be Chinese. If the instructor uses both languages, transcribe both.
Example:eng, cmn
Term Name: subtitles
URI:https://moocams.github.io/#subtitles
Label:Course Subtitles
Description:The language of subtitles provided by this course, if any.
Rule:Language is a controlled vocabulary chosen from ISO 639-3 Code.
Example:eng, cmn
Term Name: level
URI:https://moocams.github.io/#level
Label:Course Level
Description:The degree of difficulty related to subject matters.
Rule:Choose from: [Beginner | Intermediate | Advanced | Appropriate for all].
Comment:If there is no explicit level information provided by the course designers, leave it blank instead of ‘Appropriate for all’.
Example:Beginner, Intermediate, Advanced
Term Name: prerequisites
URI:https://moocams.github.io/#prerequisites
Label:Course Prerequisites
Description:The knowledge which users are expected to have before taking this course.
Rule:Use the “take what you see” approach.
Comment:The element is not repeatable. If there are more than one prerequisite, separate them using semicolons.
Example:Basic Python programming skills; Calculus; …
Term Name: courseLoad
URI:https://moocams.github.io/#courseLoad
Label:Course Load
Description:The estimated workload to achieve the course measured in hour.
Rule:It can be transcribed in two formats: (1) weekly workload: [number of hours] hour(s)/week, (2) the whole workload: [number of hours] hour(s).
Comment:If the course load is provided in another format, please try to convert it into either one of the formats mentioned in the rule. It can be transcribed in range.
Example:3-4 hours/week, 30 hours
Term Name: syllabus
URI:https://moocams.github.io/#syllabus
Label:Course Syllabus
Description:The outline or summary of the subjects to be covered in the course.
Rule:Use the “take what you see” approach for each topic and separate the topics by semicolons.
Comment:Only put the topics or titles of each unit. Do not put detailed descriptions.
Example:1. Background, Getting Started, and Nuts & Bolts; 2. Programming with R; 3. Loop Functions and Debugging; 4. Simulation & Profiling; ...
Term Name: institution
URI:https://moocams.github.io/#institution
Label:Institution
Description:The name(s) of the institution(s) from where the course comes.
Rule:Use Library of Congress Name Authority to describe its name if applicable. If the name is not available in it, use the most common form of the name.
Comment:Input all organizations that take part in the producing of this course. It is usually university name.
Example:Stanford University, National Taiwan University, Google (Firm)
Term Name: instructor
URI:https://moocams.github.io/#instructor
Label:Instructor
Description:Name(s) of the person(s) who deliver(s) this course.
Rule:Use Library of Congress Name Authority to describe its name if applicable. If the name is not available in it, use the name that appears on the course page.
Example:Charles Severance
Term Name: provider
URI:https://moocams.github.io/#provider
Label:Provider
Description:The name of the platform/website which provides this course.
Rule:Input the full official version of its name.
Comment:If the provider has more than one official name, input the name in the original language.
Example:Coursera, edX, Udacity
Term Name: certificate
URI:https://moocams.github.io/#certificate
Label:Certificate
Description:The type(s) of certificate the course provides.
Rule:The value must be chosen from: [ No Certificate | Statement of Accomplishment | Verified Certificate | Other ]. If the course provides different types of certificate based on the fee you are willing to pay, transcribe all of them separated by semicolons.
Example:No Certificate; Verified Certificate
Term Name: fee
URI:https://moocams.github.io/#fee
Label:Course Fee
Description:The fee users have to pay to access the course or to get the certificate.
Rule:The record should follow the format: [currency code] [amount] - [description]. Currency code must be chosen from ISO 4217 (http://www.iso.org/iso/home/standards/currency_codes.htm). Separate different types of fee by semicolons.
Comment:Use the original currency code.
Example:USD 0 - Statement of Accomplishment; USD 79 - Verified Certificate
Term Name: openPeriod
URI:https://moocams.github.io/#openPeriod
Label:Course Open Period
Description:The period during which the course is available or has to be completed.
Rule:Follow the format: YYYY/MM/DD - YYYY/MM/DD. Use ‘self-paced’ if the course has no end date.
Comment:Regarding ‘self-paced’ course, the starting date is prefered if available.
Example:2016/01/01 - 2016/04/30, self-paced, 2016/01/01 - self-paced
Term Name: rightsHolder
URI:https://moocams.github.io/#rightsHolder
Label:Rights Holder
Description:A person or organization owning or managing rights over the resource.
Rule:Use Library of Congress Name Authority to describe its name if applicable. If the name is not available in it, use the most common form of the name.
Example:Udacity Inc.
Term Name: tags
URI:https://moocams.github.io/#tags
Label:Tags
Description:Keywords, search terms, or terms related to the course.
Rule:Separate tags using semicolons.
Example:Web design; HTML
Term Name: rating
URI:https://moocams.github.io/#rating
Label:Course Rating
Description:The evaluation or assessment of the course on a particular scale by users.
Rule:Format: [average score] / [scale] (number of raters).
Comment:The value should be updated on a regular basis.
Example:4.5 / 5 (500)
Term Name: relatedCourse
URI:https://moocams.github.io/#relatedCourse
Label:Related Course
Description:The URI to the course which is related the transcribed course.
Rule:Input the URI from the identifier element of the related course.
Comment:The related courses are usually the ones in the same program or specialization. The ones which are closely related should also be transcribed; i.e., Algorithm part-1 and Algorithm part-2.
Example:https://www.coursera.org/learn/wharton-marketing
